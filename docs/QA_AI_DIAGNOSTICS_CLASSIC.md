# Classic: диагностика устройства и AI

Документ описывает новый сценарий в разделе `Профиль -> Проверка устройства (QA)` для классической версии.

## Что добавлено

- Поэтапная проверка строк устройства:
  - `Ожидает проверки`
  - `Проверяется` (спиннер)
  - `OK` (зелёная галочка) или `Ошибка` (красный крест)
- Блок `Диагностика AI в чате` с живой проверкой провайдеров:
  - Gemini
  - Together
  - OpenRouter
  - Grok/xAI
  - Groq
- Отображение технических деталей по провайдеру:
  - модель
  - HTTP-статус
  - краткий фрагмент ошибки
- Блок `Техническая информация для отправки в чат` и кнопка копирования отчёта.

## API диагностики

- Маршрут: `apps/classic/app/api/llm/diagnostics/route.ts`
- Метод: `GET`
- Назначение:
  - проверяет наличие ключа для каждого провайдера;
  - выполняет короткий тестовый запрос;
  - возвращает структурированный результат для UI.

## Формат результата (упрощённо)

- `checkedAt`: timestamp проверки
- `results[]`:
  - `provider`
  - `enabled`
  - `model`
  - `ok`
  - `httpStatus` (если есть)
  - `message`

## Зачем это нужно

- Быстро понять, почему не работает live-ответ в чате.
- Сократить время на разбор инцидентов: пользователь копирует готовый технический отчёт и отправляет его в чат поддержки/разработки.
