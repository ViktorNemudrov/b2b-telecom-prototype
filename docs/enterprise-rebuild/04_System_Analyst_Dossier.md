# Досье для системного аналитика (SA)

## Цель
- Перевести бизнес-требования в системные спецификации, контракты и формальные ограничения реализации.

## Текущее состояние (As-Is)
- Логика ассистента в текущей версии в значительной степени находится на фронтенде и сочетает детерминированные сценарии с live-резервированием.
- Есть прокси-эндпоинты API для LLM-провайдеров (`/api/llm/chat`, `/api/llm/gemini`).
- Значительная часть доменных данных находится в мок-моделях.
- Сильная тестовая база модульных и e2e-тестов для сценариев ассистента и навигации.

## Целевое состояние
- Разделение доменного, прикладного и интеграционного слоев.
- Явные API-контракты между фронтендом и бэкендом.
- Полный контракт данных для счетов, обращений и звонков.
- Аудитируемость и наблюдаемость обязательны на каждом критичном шаге.

## Функциональные требования (FR)
- FR-01: Аутентификация и сессия с ролевым доступом.
- FR-02: Ассистент обрабатывает запросы через конвейер intent/router/policy.
- FR-03: Сценарии счетов: список, фильтр, детализация, статус, действия.
- FR-04: Сценарии обращений: создание, фильтрация, просмотр деталей, статусы.
- FR-05: Сценарии звонков: список пропущенных, детали, статистика.
- FR-06: Настройки и кастомизация интерфейса.
- FR-07: История и телеметрия взаимодействий (без компрометации данных).

## Нефункциональные требования (NFR)
- Доступность: 99.9% для ключевого API.
- P95-задержка:
- ассистент (детерминированный) <= 1.2s,
- ассистент (live) <= 4.0s с резервным ответом <= 1.5s.
- Безопасность: базовый профиль OWASP ASVS, секреты не попадают в клиентский бандл.
- Масштабируемость: горизонтальный масштаб stateless API.
- Наблюдаемость: трассировка + структурированные логи + бизнес-метрики.

## Доменные контракты данных (начальный контур)
- Invoice:
- `id`, `amount`, `currency`, `status`, `dueDate`, `billingPeriod`, `source`.
- Appeal:
- `id`, `title`, `category`, `status`, `createdAt`, `owner`, `history`.
- Call:
- `id`, `direction`, `missed`, `time`, `counterparty`, `summary`, `recordRef`.
- AssistantMessage:
- `id`, `role`, `text`, `intent`, `sourceType`, `confidence`, `actions`.

## Базовый API-контур
- `POST /assistant/query`
- запрос: `userId`, `sessionId`, `prompt`, `context`.
- ответ: `message`, `intent`, `source`, `actions`, `next`.
- `GET /invoices`, `GET /invoices/{id}`
- `GET/POST /appeals`, `GET /appeals/{id}`
- `GET /calls/missed`, `GET /calls/{id}`

## Модель ошибок
- Единый формат ошибки:
- `code`, `message`, `details`, `traceId`, `recoverable`.
- Политика повторов/таймаутов/circuit-breaker для внешних AI-провайдеров.

## Диаграммы состояний и последовательностей (обязательны в Confluence)
- Диаграмма последовательности запроса ассистента.
- Диаграмма последовательности сценария оплаты счета.
- Диаграмма последовательности создания/обновления обращения.
- Диаграмма состояний сессии и авторизации.

## Трассировка
- Для каждого FR:
- связанный BR,
- эндпоинт API,
- сущность данных,
- экран/компонент фронтенда,
- тест-кейсы (ручные и автоматические).

## SA-чеклист (до разработки)
- FR/NFR утверждены.
- Контракты API версионированы.
- Правила валидации и каталог ошибок готовы.
- Требования безопасности и приватности включены в спецификации.
